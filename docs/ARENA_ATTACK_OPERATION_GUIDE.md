# Arena Attack 操作说明与已知问题

更新时间：2026-06-27  
适用版本：键鼠优先 Arena Attack 首版

## 1. 启动与模式切换

1. 使用 Unity 6000.4.7f1 打开 `unity/Assets/Scenes/Main.unity`。
2. 点击 Play，默认进入 `ArenaAttack`。
3. `F8`：在 `ArenaAttack` 与旧 `Training` 间切换并重新加载场景。
4. `F7`：轮换 `KeyboardMouse / Gamepad / VrStriker / XrSimulator` 控制档并重新加载场景。
5. `F9`：显示或隐藏 Arena 调试面板。

也可以使用启动参数：`-arena`、`-training`、`-gamepad`、`-vrStriker`、`-xrSimulator`。

## 2. 当前玩法流程

- 模式为单门 90 秒净计时进攻赛，玩家与蓝队队友进攻红队后卫和门将。
- 机器人在开局以及进球、被断、死球后重新发球；重置阶段不扣比赛时间。
- 没有越位、犯规、界外球、角球或任意球。
- 足球碰到场边围墙会回弹，比赛继续。
- 只有进球、对方稳定控球、球掉落/逃出场地或长时间卡死才会重置。

## 3. 键鼠操作

| 操作 | 按键 | 当前逻辑 |
|---|---|---|
| 移动 | `WASD` | 按当前摄像机水平朝向移动 |
| 视角 | 移动鼠标 | 旋转当前第三人称跟随镜头 |
| 冲刺 | `Left Shift` | 持续按住提高移动速度 |
| 主动停球 | `E` | 球在有效距离内时降低球速并短暂增强控球辅助 |
| 传球 | 鼠标右键 `RMB` | 按住蓄力、松开传球；方向来自摄像机水平 forward，接近队友方向时带轻度辅助 |
| 射门 | 鼠标左键 `LMB` | 按住蓄力、松开射门；方向来自摄像机水平 forward，接近球门方向时带轻度辅助 |
| 抢断 | `Space` | 短距离前冲并尝试把附近足球踢开，有冷却时间 |
| 暂停/释放鼠标 | `Esc` | 暂停 Arena 输入并解除鼠标锁定 |

传球、射门、停球和抢断都要求足球在动作有效距离内。没有碰到球时，调试面板会显示类似 `PASS - NO BALL` 或 `SHOT - NO BALL`。

## 4. 手柄与 VR 操作

| 控制档 | 操作 |
|---|---|
| Gamepad | 左摇杆移动、右摇杆视角、`LT` 冲刺、`A` 停球、`X` 传球、`RT` 射门、`B` 抢断 |
| VrStriker | 固定为前场射手角色；普通实体脚触球用于停球/自由触球，右扳机触球保留为传球意图，高速实体触球可形成射门 |
| XrSimulator | 在 Editor Play Mode 自动创建已有 XR Device Simulator，验证无头显时的输入和实体脚动作路由；不能代替 Quest 真机脚感测试 |

## 5. 当前已知问题（本版本只记录）

### 2026-06-27 现场反馈与回退记录

- 刚才尝试过静态镜头 / 传射目标解耦方案，但用户现场反馈不认可，相关改动已回退，不作为当前基线。
- 实际现象：视角没有变化；射门 / 传球仍然按不了。
- 下一轮不要直接重写镜头方案；先只读确认当前 Play Mode 中真实生效的相机对象、`ArenaPlayerController` 挂载状态、控制档、`GameplayEnabled`、输入事件、球动作距离判定和 `ArenaBallMotor.Execute(...)` 的返回原因。
- 只有确认“按键没有触发 / 触发但被 range 拦截 / 触发但不在 live 状态 / 触发但相机链路不是这条”之后，再做针对性修复。

### 鼠标左右键的传射效果不自然

- 代码定义目前是：左键射门、右键传球，均为按住蓄力、松开执行。
- 实际演示中，传球与射门的方向、力度、球响应和视觉区别表现比较奇怪，尚未达到可确认的操作手感。
- 当前先保留现象，不在本版本盲目调整参数或交换键位。
- 下一版本由实际试玩者补充具体复现：做了什么、预期球怎么走、实际球怎么走，再针对性修改。

### 第三人称摄像机跟踪未做好

- 当前摄像机会跟随玩家位置，并由鼠标持续旋转。
- 实际演示中的跟踪位置、构图和观察稳定性还不理想，不能视为最终镜头方案。
- 当前版本不继续调跟随参数，下一版优先改为静态 3D 比赛镜头。

## 6. 下一版静态镜头计划

- 使用透视相机，不做正交俯视图。
- 相机固定在对方球门上方并略微位于球门后方，朝球场中心及对面半场观察。
- 相机位置和朝向基于 `FieldBuilder` 的本地坐标计算，球场移动或旋转后仍保持正确构图。
- Arena 比赛期间不跟随玩家、不自动旋转、不抖动；鼠标不再控制比赛镜头。
- `WASD` 继续按静态镜头的水平投影方向移动，保持屏幕方向直觉。
- 必须把传射瞄准与摄像机 forward 解耦：静态镜头朝向本方半场，不能直接作为射门方向。
- 第一版拆分后的默认瞄准建议：射门以“玩家到对方球门”为基准，传球以“玩家到队友”为基准，抢断保持玩家自身 forward；后续再决定是否加入鼠标落点瞄准。

静态镜头验收标准：玩家、球、队友、后卫和球门主要区域在 16:9 画面中持续可见；比赛过程中镜头位置不变化；球门结构和围墙不遮挡主要进攻区域；传射方向不受镜头朝向反转影响。
