# SoccerBot 核心玩法 V1.2 演示强化计划

日期：2026-06-07

## 目标

让玩家和评委在不听额外讲解的情况下，也能立刻看懂：

球来了，我要接；接得好，就有更好的射门机会；接得差，还能立刻反抢。

## 当前问题

核心玩法已经有接球判定、接球质量、反抢和射门结算，但演示时存在一个关键断点：

玩家在机器人/队友发球时，并不知道自己需要主动按键接球。

这会导致球飞向玩家时，被理解成普通演出动画，而不是一个需要参与的技能窗口。核心玩法已经存在，但没有被画面提示说清楚。

## V1.2 体验要求

玩家必须在 1 秒内知道：

- 球正在传给我
- 我需要准备接球
- 应该按哪个键
- 什么时候按最好
- 按得好不好会影响后续射门

## 演示优先级

### P0：来球阶段强提示

在 `Pass` 阶段开始时显示明确提示。

PC 文案：

- `准备接球`
- `按 Space / 鼠标左键`

Quest 文案：

- `准备接球`
- `握紧 Grip 或按 Trigger`

提示形式：

- 屏幕中央偏下，或世界空间玩家正前方
- 文字不要太长
- 出现时间覆盖整个传球飞行阶段
- 接球成功、失败或 Pass 阶段结束时立即消失

设计要求：

- 这不是教学弹窗，而是比赛中的即时战术提示
- 不阻挡球和球门
- VR 内必须可见，不能只依赖 `ScreenSpaceOverlay`

### P0：接球窗口节奏提示

在球飞行过程中给玩家一个“现在快按”的节奏信号。

状态建议：

- 窗口未到：白色/蓝色，`准备`
- 进入可接窗口：高亮，`接球！`
- 接近完美点：短促闪光或放大
- 窗口错过：红色，`错过接球`

第一版可以先用最轻量实现：

- 一个 `TextMeshProUGUI` 或世界空间 TMP 文本
- 根据 `_passProgress01` 映射显示状态
- 暂不做复杂模型、粒子或节奏小游戏

### P1：接球落点圈

在玩家脚边或球预计到达点显示接球圈。

表现建议：

- 球飞来时出现圆环
- 圆环随着接球窗口临近逐渐发亮
- 完美窗口附近短暂收缩/闪光
- 接球失败时变红并快速消失

第一版技术选择：

- 简单圆环 mesh
- 或 `LineRenderer`
- 或扁平 Cylinder/Quad + 透明材质

目标不是做华丽特效，而是让玩家知道：球要到这里，我要在这里接。

### P1：接球质量反馈

玩家按下接球键后，立即显示质量反馈。

中文反馈：

- `完美停球`
- `稳定接球`
- `时机过早`
- `时机偏晚`
- `朝向错误`
- `停球失败`

反馈规则：

- 高质量：绿色/金色，进入持球，后续射门有加成
- 中质量：蓝色/白色，正常持球
- 低质量：红色，进入反抢
- 未按键：显示 `没有接球` 或 `错过接球`

这一步的目标不是评分精细，而是让玩家明白自己刚才做了什么。

### P1：结算中解释接球影响

射门或回放评分时，展示接球质量对结果的影响。

文案建议：

- `接球质量：优秀，射门机会提升`
- `接球质量：一般，射门机会正常`
- `接球质量：较差，被对手压迫`

如果显示数值：

- `First Touch +18%`
- `Poor Touch -12%`

中文演示优先使用直观描述，数值作为辅助。

## 推荐新增组件

### `ReceptionPromptPresenter`

职责：

- 管理 Pass 阶段提示文字
- 根据传球进度切换 `准备 / 接球 / 错过`
- 兼容 PC 和 Quest 文案
- 阶段结束时隐藏

建议路径：

`Assets/Scripts/UI/ReceptionPromptPresenter.cs`

### `ReceptionTargetIndicator`

职责：

- 在玩家脚边或接球目标点显示接球圈
- 根据接球窗口改变颜色、亮度、缩放
- 接球成功/失败后播放短暂反馈并消失

建议路径：

`Assets/Scripts/Flow/ReceptionTargetIndicator.cs`

如果实现偏 Canvas/TMP，可以放到 `Assets/Scripts/UI/`。

### `ReceptionFeedbackPresenter`

职责：

- 显示接球质量结果
- 将 `ReceptionRules` 计算结果转成玩家可读的反馈等级
- 为 `ScorePanel` 或后续结算提供显示文本

第一版不要引入复杂事件总线。可以由 `MatchFlowController` 在接球后直接调用。

## MatchFlowController 接入点

### `DoPass()`

需要加入：

- Pass 开始：显示接球提示和接球圈
- 每帧：更新提示状态和接球圈状态
- Pass 结束：如果未接球，显示 `没有接球 / 错过接球`

### `HandleReceiveAttempt()`

需要加入：

- 根据 `_receiveQuality` 显示接球反馈
- 成功时隐藏接球提示
- 失败时切到红色反馈，为 Recovery 做铺垫

### `DoPossession()`

需要加入：

- 如果进入 Recovery，提示文案从 `接球` 切换为 `反抢`
- 如果成功持球，显示短促正反馈，例如 `控球成功`

## 第一轮实现顺序

1. 新增 `ReceptionPromptPresenter`
2. 在 `DoPass()` 显示 `准备接球 / 接球！`
3. 在 `HandleReceiveAttempt()` 显示接球质量反馈
4. 未按键时显示 `错过接球`
5. 确认 PC Editor 可读
6. 再适配 Quest 世界空间显示
7. 之后再做接球落点圈

## 暂时不做

- 不做复杂新手教程
- 不做长篇说明文字
- 不做完整节奏小游戏
- 不做脚部追踪
- 不把 SmartBall/BLE 接入这个提示系统

## 验收标准

PC Editor：

- 球飞出时，玩家能看到 `准备接球`
- 接球窗口到来时，提示明显变成 `接球！`
- 玩家按键后，立即看到接球质量
- 玩家不按键，能看到 `错过接球`
- 进入反抢前，玩家知道是因为停球失败

Quest：

- 提示位于玩家正前方或舒适视野内
- 文字不贴脸、不遮挡球
- Grip/Trigger 文案清楚
- 玩家第一次体验也能知道 Pass 阶段要主动接球
